How to Use Roadmap

System Design interviews are composed of three main sections. Real life problems, where you will be given 30 to 45 minutes to complete a working design. Trivia, where an interviewer will ask you to explain a variety of database, server, API, networking, and other topics. And finally, you can be asked "Low level design", which tests your ability to code and use object oriented programming design patterns.

In my experience, I've had interviews where they will test knowledge in multiple of these areas. For that reason, if I want to be as prepared as possible, it's a requirement to know each of these main topics. Because of this, I've broken this roadmap into three main pages for each of the topics.
