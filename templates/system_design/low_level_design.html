Low Level Design (Object Oriented Programming)

Master object-oriented design patterns and practice coding system components. These problems test your ability to write clean, maintainable, and extensible code.

pattern Essential Design Patterns

single_bed Singleton Pattern

Ensures only one instance of a class exists throughout the application lifetime.

Global Access Lazy Initialization Thread Safety
Use cases: Database connections, loggers, configuration managers
visibility Observer Pattern

Defines a one-to-many dependency between objects so that when one object changes state, all dependents are notified.

Event Handling Loose Coupling Notifications
Use cases: GUI event handling, model-view architectures
build Factory Pattern

Creates objects without specifying the exact class to create. Useful for creating families of related objects.

Object Creation Abstraction Flexibility
Use cases: UI components, database drivers, file parsers
layers Decorator Pattern

Adds new functionality to objects dynamically without altering their structure.

Dynamic Behavior Composition Extension
Use cases: Stream processing, UI decorations, middleware
psychology Strategy Pattern

Defines a family of algorithms, encapsulates each one, and makes them interchangeable.

Algorithm Selection Runtime Choice Encapsulation
Use cases: Payment processing, sorting algorithms, compression
published_with_changes Command Pattern

Encapsulates a request as an object, allowing you to parameterize clients with different requests, queue operations, and support undo.

Undo/Redo Queuing Logging
Use cases: Text editors, remote controls, macro recording

rule SOLID Principles

S
Single Responsibility Principle

A class should have only one reason to change

O
Open/Closed Principle

Open for extension, closed for modification

L
Liskov Substitution Principle

Objects should be replaceable with their subtypes

I
Interface Segregation Principle

Many specific interfaces are better than one general

D
Dependency Inversion Principle

Depend on abstractions, not concrete implementations
